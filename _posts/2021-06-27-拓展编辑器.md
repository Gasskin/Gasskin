### 菜单

```csharp
[MenuItem("Assets/MyTools/Tool1",false,2)]
private static void MyTools1()
{
    Debug.Log("right mouse");
}

[MenuItem("Assets/MyTools/Tool2",false,1)]
private static void MyTools2()
{
    Debug.Log("right mouse");
}
```

1. 第一个参数为所在位置，可以添加快捷键，如：

   ```csharp
   _w 单一的快捷键 W 
   #w shift+w 
   %w ctrl+w 
   &w Alt+w
   ```

2. 第二个参数决定了这个方法是否是一个“验证函数”，如果这个方法A是“验证函数”，那么这个方法A会作为另一个**同位置**方法B的先验方法，只有当方法A返回值为true时，才会执行方法B，如下：

   ```csharp
   [MenuItem("GameObject/Extend/Test1",true)]
   private static bool ValidateTest1()
   {
       return Selection.activeObject != null;
   }
   
   [MenuItem("GameObject/Extend/Test1",false)]
   private static void Test1()
   {
       Debug.Log(Selection.gameObjects.Length);
   }
   ```

   你会发现这两个方法的路径名称是一模一样的，而第一个方法的第二个参数为true，说明第一个方法是第二个方法的先验方法，如果是这么写的话，当你没有选中任何物体也就是先验方法返回false时，第二方法会是灰色的，无法点击，只有当你选中某一个物体时，才能够使用第二个方法

3. 第三个参数是排序用的，优先级

### 布局

```csharp
[InitializeOnLoadMethod]
static void Test2()
{
    EditorApplication.projectWindowItemOnGUI += ((guid, rect) =>
    {
        if (Selection.activeObject && guid.Equals(AssetDatabase.AssetPathToGUID(AssetDatabase.GetAssetPath(Selection.activeObject))))
        {
            float w = 50f;
            rect.x += rect.width - w;
            rect.y += 2f;
            rect.width = w;
            GUI.color=Color.red;
            if (GUI.Button(rect,"Click"))
            {
                Debug.Log(Selection.activeObject.name);
            }
            GUI.color=Color.white;
        }
    });
}
```

1. [InitializeOnLoadMethod]代表此方法会在代码编译完成后首先被调用

2. EditorApplication.projectWindowItemOnGUI 是一个委托，会监听Project面板的变化，我们可以写委托来监听，此方法监听所有Projcet面板的变动，并且只会在我们当前选中的目标后加入一个红色点击按钮

   ![1](https://www.logarius.icu/images/UnityStudy/Unity3D/1.png)

3. 上述委托有两个参数，第一个参数是选中目标的guid，第二个参数是选中的目标区域，我们需要利用目标区域来绘制点击按钮的大小，在unity中，区域有4个属性即（x,y,width,height），其中xy是区域左上角的坐标，width代表宽度，height代表高度

   ![1](https://www.logarius.icu/images/UnityStudy/Unity3D/2.png)

   其中背景为蓝色的就是当前选中的区域，根据这个区域的4个属性，我们很容易的可以绘制出一个想要的按钮
