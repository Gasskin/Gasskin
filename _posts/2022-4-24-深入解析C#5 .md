# 异步函数

由**async**修饰的方法或者lambda表达式，它可以使用await运算法

```c#
public class Program
{
    private static HttpClient client = new HttpClient();
    
    static void Main(string[] args)
    {
        Log();
        Console.WriteLine("test");
        Console.ReadKey();
    }

    private static async void Log()
    {
        Console.WriteLine("loading...");
        var text = await client.GetStringAsync("https://www.baidu.com");
        Console.WriteLine(text.Length);
        Console.WriteLine("end");
    }
}
```

结果如下:

```c#
loading...
test
9193
end
```

开启异步的线程还是主线程，但很明显异步任务不是在主线程中执行的，原因有2

1. 没有阻塞主线程
2. 结束时的线程不是主线程

也可以把await和方法分开调用

```c#
public class Program
{
    private static async void Log()
    {
        Console.WriteLine("loading...");
        var task =  client.GetStringAsync("https://www.baidu.com");
        var text = await task;
        Console.WriteLine(text.Length);
        Console.WriteLine("end");
    }
}
```

需要注意，此时**task**变量是**Task\<string>**类型的，而**await task**的类型却是**string**

异步的关键在于，当遇到**await**表达式时，方法会立刻返回，随后检查**await**之后的表达式是否已经有结果，如果没有，则开辟一个新的线程等待表达式执行结束，之后继续执行，本质也是一种回调

# async方法声明

声明很简单，只要在方法前加上async关键字

```c#
public async void Test();
```

## 返回值

目前只能是以下三种类型之一

1. **void**
2. **Task**
3. **Task\<T>**

## 参数

比较随意，但不能由ref和out修饰

# await表达式

使用async修饰的方法可以在内部调用await表达式

同时await的优先级比"."小，所以可以链式调用

```c#
var result = await forr.Bar().Baz();
```

## 可等待模式

await后接着的表达式，必须是可等待的

假设一个返回类型为T的表达式，他需要是可等待的，那么T必须满足

1. T必须具备（可以是扩展方法）无参方法GetAwaiter()，此方法返回一个awaiter类型
2. awaiter类型必须实现INotifyCompletion接口，此接口只有一个方法：void OnCompleted(Action)
3. awaiter类型必须具有一个可读的实例属性IsCompleted，是bool
4. awaiter类型必须具有一个非泛型的无参方法GetResult()，await关键字的返回值会和此方法保持一致

简单来说就是这个类T应该类似如下形式

```c#
public struct YieldAwaitable
{
    // 一个返回等待器的实例方法
    public YieldAwaiter GetAwaiter()
    {
        return new YieldAwaiter();
    }
        
    // 具体的等待器，实现接口
    public struct YieldAwaiter : INotifyCompletion
    {
        public bool IsCompleted {get;}

        public string GetResult()
        {
            return "result";
        }
            
        public void OnCompleted(Action continuation)
        {
        }
    }
}
```

然后就可以进行如下调用了

```c#
private static async void Test()
{
    Console.WriteLine("start");
    var str = await new YieldAwaitable(); // 这里合法因为上面的GetResult返回string
    Console.WriteLine(str);
}
```

# 返回值的封装

```c#
internal class Program
{
    private static HttpClient client = new HttpClient();
    
    static void Main(string[] args)
    {
        var length = Test();
    }

    private static async Task<int> Test()
    {
        var task = client.GetStringAsync("https://www.baidu.com");
        await task;
        return task.Result.Length;
    }
}
```

异步方法的返回值类型是Task\<int>，但是我们返回的确是一个int类型

实际上编译器为我们进行了封装，最终调用Test方法，得到的还是Task\<int>类型

如果一个异步方法返回的只是Task，那么与void是一样的
